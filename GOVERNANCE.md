# Governance

SimuLoom currently uses a maintainer-led governance model. The repository owner is responsible for
release decisions, security response, compatibility policy, and final merge authority.

Technical decisions favor deterministic behavior, backward compatibility, secure defaults,
portable formats, and evidence-backed operation. Significant changes should be proposed publicly
and evaluated against those principles. Contributors earn broader review responsibility through
sustained, constructive participation and demonstrated care for the project's security boundary.

This model is expected to evolve if the maintainer and contributor community grows.
